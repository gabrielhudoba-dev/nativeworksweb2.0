SERVICE SCHEDULE — WEB DESIGN & DEVELOPMENT

Annexed to SOW Ref: {{sow_ref}}

─────────────────────────────────────────────────

W1. SCOPE

This Schedule covers the design and development of web properties as described in the Proposal, including but not limited to: information architecture, visual design (desktop and mobile), front-end development, content management system integration, and pre-launch quality assurance.

W2. DELIVERABLES

The following deliverables are included in the Fees under this Schedule:

     (a) Wireframes / low-fidelity prototypes for agreed page types
     (b) High-fidelity visual designs (responsive: desktop, tablet, mobile)
     (c) Developed and tested codebase per the agreed technology stack
     (d) Integration with specified third-party services (as listed in Proposal)
     (e) Handover documentation and codebase transfer
     (f) One (1) round of post-launch bug fixes within thirty (30) days of launch

W3. NOT INCLUDED (REQUIRES CHANGE ORDER)

The following are explicitly excluded unless added by Change Order:

     — Content copywriting or translation
     — SEO strategy or keyword research beyond technical SEO implementation
     — Custom illustration or photography
     — Hosting, domain registration, or ongoing maintenance (see Retainer Schedule if applicable)
     — Database architecture beyond what is specified in the Proposal
     — Third-party API integration fees or licences
     — Browser compatibility beyond: latest two versions of Chrome, Firefox, Safari, Edge

W4. TECHNOLOGY STACK

Stack: As specified in the Proposal or as agreed in writing between the Parties.

W5. REVISION ROUNDS

Web design revisions: Two (2) rounds of revisions per page type are included.
Development: Bug fixes identified during a seven (7) business day QA window post-delivery are included. Post-launch bugs are addressed under Section W2(f).

W6. BROWSER AND DEVICE COMPATIBILITY

Native Works shall test and ensure compatibility with the two most recent versions of: Google Chrome, Mozilla Firefox, Apple Safari, and Microsoft Edge. Testing shall cover desktop, tablet (768px+), and mobile (375px+) viewports.

W7. ACCEPTANCE

Client shall conduct a final review of the developed site/application and provide written approval or a consolidated list of specific defects within five (5) business days of delivery of access credentials. Approval is deemed given after this period.

W8. POST-LAUNCH WARRANTY

Native Works provides a thirty (30) day post-launch warranty covering defects that cause the site/application to function materially differently from the agreed specification. This warranty does not cover changes to requirements, third-party service failures, or issues arising from Client modifications.

W9. HANDOVER

Upon final payment, Native Works shall deliver:
     — Complete source code via agreed repository or file transfer
     — Access credentials for all accounts created on Client's behalf
     — Handover documentation covering deployment, CMS usage, and key configurations
     — Transfer of ownership of third-party accounts created specifically for this project
